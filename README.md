# GC-IMS Autoencoder Augmentation Pipeline

Transformer-autoencoder pipeline for generating and evaluating synthetic GC-IMS spectra. The project trains two sequential autoencoders, fits class-conditional Gaussian latent models, decodes synthetic latent samples back into spectra, and evaluates whether the generated spectra are visually and quantitatively close to real measurements.

The current implementation is conservative for a small fermentation GC-IMS dataset:

* native `.mea` loading through `gc-ims-tools`
* train-only preprocessing window fitting
* RIP alignment and optional RIP-region removal
* shared peak-aware cropping and resizing to a fixed model shape
* `log1p` style intensity compression and reversible scaling
* sequential column and row transformer autoencoders
* class-conditional Gaussian latent sampling with covariance shrinkage
* classifier and spectral-similarity diagnostics for synthetic data validation

```powershell

.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m pip install -e .
```

Beast / Linux:

```bash
cd ~/autoencoder_pipeline
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .
```

## Data

The default data path is still `data_fermentation/` for compatibility with the current lab-server setup. You can override it without editing code:

```bash
export GCIMS_DATA_PATH=~/autoencoder_pipeline/data_fermentation
```

Recommended long-term layout:

    data_fermentation/
      GCIMS_mixed_cultures/
        E. coli and S. cerevisiae/
          ...
      GCIMS_pure_cultures/
        E. coli/
          ...

Large data should remain outside git. If you prefer the cookiecutter layout, place the raw dataset under `data/raw/` and point `GCIMS_DATA_PATH` at that folder.

## Preprocessing

The preprocessing path is:

1. load vendor-preprocessed `.mea` spectra
2. split into train and validation sets
3. fit the shared preprocessing window on train spectra only
4. detect and align the RIP column
5. optionally remove the RIP-dominated region
6. crop around the shared informative ROI
7. resize to `MAX_MODEL_ROWS x MAX_MODEL_COLS`
8. compress and scale intensities for autoencoder training

This is intentionally not a word-for-word copy of the reference paper. The current version is built around stable windows, conservative latent sampling, and diagnostic visibility for small GC-IMS datasets.

## Evaluation

The classifier table reports three core cases:

* `80% real -> 20% real`: baseline downstream classifier behavior
* `80% real + 100% synthetic -> 20% real`: whether synthetic data helps real-test performance
* `100% real -> 100% synthetic`: whether generated samples land in real learned class regions

The spectral-similarity table complements classification with direct spectrum-level checks:

* cosine similarity
* Pearson correlation
* global SSIM
* MAE and RMSE
* intensity quantile distance
* nearest-neighbor PCA distance
* peak-mask IoU
* embedding MMD and centroid distance

Interpret the results conservatively. If augmentation does not improve real-test accuracy, but real-to-synthetic classification and spectral similarity are reasonable, the generated spectra may still be useful as a plausibility or robustness check rather than a guaranteed classifier boost.
